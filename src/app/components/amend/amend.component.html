git commit amend - how it works ?

Git amend est utilisé pour apporter des modifications à la dernière validation (commit) en combinant les
modifications avec le commit précédent, permettant ainsi de corriger des erreurs, d'ajouter des fichiers oubliés,
ou de mettre à jour le message du commit, sans créer un nouveau commit distinct.
Cela contribue à maintenir un historique de version plus propre et plus clair.

1. Select Files to commit in IntelliJ

2. Select "Amend" to commit

3. Click on Commit

4. Do $ git push --force or $ git -f

5. $ git checkout .

6. $ git checkout develop

7. $ git pull et git fetch prune ou git -fp

8. $ git checkout maBranche

9. $ git rebase develop

10. $ git push --force

Note: hors entreprise, on fait un $ git add et un $ git commit --amend et un $ git push origin --force pour commit sur le dernier commit (HEAD)
